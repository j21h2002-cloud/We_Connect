# 카카오 알림톡 발송 시스템 — 설치 및 세팅 가이드

> **대상**: 처음 PC를 세팅하는 담당자
> **난이도**: 기초 (IT 지식 없어도 가능)
> **총 소요 시간**: 약 1~2시간 (카카오 심사 제외)

---

## 전체 흐름 한눈에 보기

```
[1단계] 프로그램 설치        → VS Code, Git, Node.js 설치
[2단계] 시스템 파일 준비     → 폴더 복사 또는 다운로드
[3단계] 로컬 테스트          → 내 PC에서 데모 모드 동작 확인
[4단계] Supabase 세팅        → 클라우드 데이터베이스 만들기
[5단계] GitHub 업로드        → 소스코드 클라우드에 올리기
[6단계] Netlify 배포         → 실제 웹사이트 주소 만들기
[7단계] Aligo + 카카오 채널  → 알림톡 발송 서비스 신청
[8단계] 시스템 최종 설정     → URL·키 입력, 계정 등록
```

## 1단계. 필수 프로그램 설치

> ⏱ 소요 시간: 약 20분

### 1-1. VS Code 설치

VS Code는 시스템 파일을 수정할 때 사용하는 편집기입니다.

1. 브라우저에서 `https://code.visualstudio.com` 접속
2. 파란색 **"Download for Windows"** 버튼 클릭
3. 다운로드된 `.exe` 파일 실행 → **"동의합니다"** 체크 → **"다음"** 계속 클릭 → **"설치"**
4. 설치 완료 후 **"Visual Studio Code 실행"** 체크 상태로 **"마침"**

### 1-2. Live Server 확장 설치 (로컬 테스트용)

1. VS Code 실행
2. 왼쪽 사이드바에서 **블록 아이콘(Extensions)** 클릭 또는 `Ctrl + Shift + X`
3. 검색창에 **"Live Server"** 입력
4. 목록 첫 번째 **"Live Server"** (Ritwick Dey 작성) → **"Install"** 클릭

### 1-3. Git 설치 (GitHub 업로드에 필요)

1. 브라우저에서 `https://git-scm.com` 접속
2. **"Download for Windows"** 클릭
3. 다운로드된 `.exe` 실행 → 모든 화면에서 **"Next"** 클릭 (기본값 유지) → **"Install"**

### 1-4. Node.js 설치 (Netlify 배포에 필요)

1. 브라우저에서 `https://nodejs.org` 접속
2. **"LTS"** 버전 다운로드 클릭
3. 다운로드된 `.msi` 파일 실행 → **"Next"** 계속 → **"Install"**
4. 설치 완료 후 확인:
   - `Windows 키 + R` → `cmd` 입력 → Enter
   - `node --version` 입력 → `v20.x.x` 같은 버전 번호가 나오면 성공

## 3단계. 로컬 테스트 (데모 모드)

> ⏱ 소요 시간: 약 5분
> 실제 인터넷 서버 없이 내 PC에서 기능을 미리 확인합니다.

1. VS Code 실행 → **"파일 > 폴더 열기"** → `카카오_알림톡_웹` 폴더 선택
2. 왼쪽 파일 목록에서 `login.html` 클릭
3. VS Code 우하단의 **"Go Live"** 버튼 클릭
4. 브라우저가 자동으로 열리면서 로그인 화면 표시

### 데모 로그인 계정

| 구분 | 아이디 | 비밀번호 |
|------|--------|----------|
| 관리자 | `admin` | `admin123` |
| 일반사용자 | `user1` | `user123` |

> ⚠️ 데모 모드에서는 데이터가 저장되지 않습니다. 새로고침하면 초기화됩니다.
> 실제 운영은 4단계 이후 설정이 완료되어야 합니다.

## 4단계. Supabase 세팅 (클라우드 데이터베이스)

> ⏱ 소요 시간: 약 30분
> 고객사 정보, 발송 이력 등 모든 데이터가 저장되는 곳입니다.

### 4-1. 가입

1. 브라우저에서 `https://supabase.com` 접속
2. 우상단 **"Start your project"** 클릭
3. **"Continue with GitHub"** 또는 이메일로 회원가입
   (GitHub 계정이 없다면 이메일 가입 후 진행)

### 4-2. 새 프로젝트 만들기

1. 로그인 후 **"New project"** 버튼 클릭
2. 아래와 같이 입력:
   - **Name**: `kakao-alimtalk` (아무 이름이나 가능)
   - **Database Password**: 기억하기 쉬운 비밀번호 입력 (나중에 필요)
   - **Region**: **Northeast Asia (Seoul)** 선택 ← 반드시 한국 선택!
3. **"Create new project"** 클릭 → 약 1~2분 기다리기

### 4-3. API 키 복사

1. 왼쪽 메뉴에서 **"Project Settings"** (톱니바퀴 아이콘) 클릭
2. **"API"** 메뉴 클릭
3. 아래 두 값을 **메모장에 복사** 해두기:
   - **Project URL**: `https://abcdefghij.supabase.co` 형태
   - **anon public**: `eyJhbGci...` 로 시작하는 긴 문자열

### 4-4. 데이터베이스 테이블 만들기

1. 왼쪽 메뉴에서 **"SQL Editor"** 클릭
2. 가운데 화면에 빈 편집창이 열림
3. `카카오_알림톡_웹/supabase/schema.sql` 파일을
   VS Code에서 열고 → `Ctrl + A` → `Ctrl + C` (전체 복사)
4. Supabase SQL Editor 창에 `Ctrl + V` (붙여넣기)
5. 우상단 **"Run"** 버튼 클릭
6. 하단에 **"Success"** 메시지가 나오면 완료

### 4-5. 이메일 인증 비활성화

> 사용자 추가 시 이메일 인증 없이 바로 로그인 가능하게 설정

1. 왼쪽 메뉴 **"Authentication"** 클릭
2. 상단 탭 **"Providers"** 클릭
3. **"Email"** 항목 클릭
4. **"Confirm email"** 토글을 **OFF** (회색)으로 변경
5. **"Save"** 클릭

## 5단계. GitHub 업로드

> ⏱ 소요 시간: 약 15분
> 시스템 파일을 클라우드에 올려 Netlify에서 배포할 수 있게 합니다.

### 5-1. GitHub 가입

1. `https://github.com` 접속 → **"Sign up"** 클릭
2. 이메일, 비밀번호, 아이디 입력 후 가입 완료
3. 이메일 인증 후 로그인

### 5-2. 새 저장소(Repository) 만들기

1. GitHub 로그인 후 우상단 **"+"** → **"New repository"** 클릭
2. **Repository name**: `kakao-alimtalk` 입력
3. **Private** 선택 (사내 시스템이므로 비공개 권장)
4. **"Create repository"** 클릭

### 5-3. 파일 업로드

1. 생성된 저장소 페이지에서 **"uploading an existing file"** 링크 클릭
2. `카카오_알림톡_웹` 폴더 안의 **모든 파일과 폴더**를
   드래그 앤 드롭으로 업로드
3. 하단 **"Commit changes"** 클릭

> ⚠️ 폴더 구조가 그대로 유지되어야 합니다.
> 폴더가 많다면 Git 명령어 방식(아래 참고)을 사용하세요.

**Git 명령어 방식 (폴더가 많을 때):**
```
1. Windows 키 → "Git Bash" 검색 → 실행
2. 아래 명령어 순서대로 입력 (GitHub 아이디·저장소명 수정):

cd "C:/Users/웹케시 이웅귀 외부망/Desktop/바탕화면/카카오_알림톡_웹"
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/[GitHub아이디]/kakao-alimtalk.git
git push -u origin main

3. GitHub 아이디·비밀번호 입력
```

## 6단계. Netlify 배포 (웹사이트 주소 만들기)

> ⏱ 소요 시간: 약 20분
> 브라우저에서 접속 가능한 실제 웹주소를 만듭니다.

### 6-1. Netlify 가입

1. `https://netlify.com` 접속 → **"Sign up"** 클릭
2. **"Continue with GitHub"** 선택 → GitHub 계정으로 로그인 허용

### 6-2. 사이트 배포

1. 로그인 후 **"Add new site"** → **"Import an existing project"** 클릭
2. **"Deploy with GitHub"** 클릭
3. 저장소 목록에서 **"kakao-alimtalk"** 선택
4. 설정은 건드리지 않고 바로 **"Deploy site"** 클릭
5. 약 1~2분 후 배포 완료 → 자동으로 URL 생성
   (예: `https://peaceful-kataoka-abc123.netlify.app`)

### 6-3. 도메인 이름 변경 (선택)

> 자동 생성된 URL이 복잡하면 변경 가능

1. **"Site settings"** → **"Domain management"**
2. **"Options"** → **"Edit site name"**
3. 원하는 이름 입력 (예: `webcash-alimtalk`)
4. 변경 후 URL: `https://webcash-alimtalk.netlify.app`

## 7단계. Aligo 가입 및 카카오 채널 개설

> ⏱ 소요 시간: **2~4주** (카카오 심사 기간 포함)
> 실제 알림톡 발송을 위해 반드시 필요합니다.

### 7-1. 알리고 가입

1. `https://www.aligo.in` 접속 → 회원가입
2. **사업자 회원**으로 가입 (사업자등록번호 필요)
3. 서비스 신청 → **"카카오 알림톡"** 선택

### 7-2. 카카오 비즈니스 채널 개설 (약 2~4주 소요)

1. `https://business.kakao.com` 접속 → 카카오 계정으로 로그인
2. **"카카오톡 채널"** → **"채널 만들기"**
3. 채널 이름, 설명, 카테고리 입력 후 개설 신청
4. 카카오 심사 완료 대기 (영업일 기준 5~10일)
5. 심사 통과 후 알리고에 **발신 프로필 등록** 요청

### 7-3. 메시지 템플릿 등록 (약 3~5 영업일 소요)

1. 알리고 또는 카카오 비즈니스에서 **템플릿 등록**
2. 발송할 메시지 내용 작성 (카카오 양식에 맞춰야 함)
3. 심사 완료 후 **템플릿 코드** 발급

### 7-4. 발급받은 정보 확인

알리고 가입 및 카카오 채널 개설 완료 후 아래 4가지 정보를 확인:

| 항목 | 설명 | 위치 |
|------|------|------|
| App Key | 알리고 발급 앱 키 | 알리고 마이페이지 |
| Sender Key | 카카오 발신 프로필 키 | 알리고 카카오 설정 |
| 알리고 아이디 | 알리고 로그인 아이디 | 알리고 가입 시 설정 |
| 발신 번호 | 문자 발신번호 | 알리고 발신번호 관리 |

## 8단계. 시스템 최종 설정

> ⏱ 소요 시간: 약 15분
> 앞에서 준비한 키 값들을 시스템에 입력합니다.

### 8-1. config.js 파일 수정 (Supabase 연결)

1. VS Code에서 `js/config.js` 파일 열기
2. 파일 맨 위 두 줄을 **4단계에서 복사한 값**으로 수정:

```javascript
// 수정 전
const SUPABASE_URL  = 'https://YOUR_PROJECT_ID.supabase.co'
const SUPABASE_ANON = 'YOUR_ANON_KEY'

// 수정 후 (본인 값으로 변경)
const SUPABASE_URL  = 'https://abcdefghij.supabase.co'   ← Project URL 붙여넣기
const SUPABASE_ANON = 'eyJhbGci...'                       ← anon public 붙여넣기
```

3. `Ctrl + S` 저장

### 8-2. GitHub에 변경사항 반영 (Netlify 자동 재배포)

**Git Bash에서 실행:**
```bash
cd "C:/Users/[사용자이름]/Desktop/카카오_알림톡_웹"
git add js/config.js
git commit -m "Supabase 연결 설정"
git push
```

> 약 1~2분 후 Netlify가 자동으로 재배포됩니다.

### 8-3. 웹사이트 접속 확인

1. Netlify에서 발급받은 URL로 브라우저 접속
   (예: `https://webcash-alimtalk.netlify.app/login.html`)
2. 로그인 화면이 정상 표시되는지 확인

### 8-4. 최초 관리자 계정 만들기

> ⚠️ 실제 Supabase 연결 후에는 데모 계정으로 로그인 불가
> 아래 방법으로 관리자 계정을 먼저 만들어야 합니다.

**Supabase 대시보드에서 계정 생성:**
1. Supabase → **"Authentication"** → **"Users"** 탭
2. **"Invite user"** 버튼 클릭
3. 관리자 이메일 형식으로 입력:
   `admin@kakao.system` (아이디가 "admin"인 경우)
4. **"Send invitation"** 클릭
5. Supabase SQL Editor에서 프로필 직접 삽입:
   ```sql
   INSERT INTO user_profiles (id, username, name, role)
   SELECT id, 'admin', '관리자', 'admin'
   FROM auth.users
   WHERE email = 'admin@kakao.system';
   ```

**또는** 웹사이트 접속 후 관리자 계정으로 로그인 →
**⚙️ 설정 > 사용자 관리 > 추가** 버튼으로 직원 계정 생성

### 8-5. 카카오 API 키 입력

1. 관리자 계정으로 로그인
2. **⚙️ 설정** 메뉴 클릭
3. **카카오 API 설정** 섹션에 아래 값 입력:
   - API Key (알리고 App Key)
   - Sender Key (카카오 발신 프로필 키)
   - 알리고 아이디
   - 발신 번호
4. **"저장"** 클릭

### 8-6. 메시지 템플릿 등록

1. **메시지** 메뉴 클릭 → **"+ 메시지 추가"** 버튼
2. 카카오에서 심사 승인된 템플릿 정보 입력:
   - 메시지 제목 (내부 관리용)
   - 템플릿 코드 (카카오 승인 코드)
   - 메시지 내용
3. **"저장"**

### 8-7. 고객사 데이터 등록

1. **고객사** 메뉴 → **"Excel 양식 다운로드"**
2. 양식에 고객사 데이터 입력 후 **"Excel 가져오기"** 로 일괄 등록

## 자주 묻는 질문 (FAQ)

**Q. 로그인이 안 돼요**
→ Supabase config.js URL·키가 올바른지 확인
→ Supabase Authentication > Users 에서 계정이 있는지 확인

**Q. 발송 버튼을 눌러도 아무 반응이 없어요**
→ 설정에서 App Key, Sender Key 입력 여부 확인
→ 알리고 계정에 충전금이 있는지 확인

**Q. 고객사가 조회되지 않아요**
→ Supabase 테이블이 정상 생성되었는지 확인 (schema.sql 재실행)

**Q. 새 PC에서 접속하려면?**
→ 별도 설치 불필요. 브라우저에서 Netlify URL만 접속하면 됩니다.

**Q. 데이터가 사라졌어요**
